Pick N' Roll

Basketball Court and Game Finder

Project Overview

This was my first full stack web application. This was an idea I have had for a while. There have been many times I wanted to play basketball but didn't know where people were playing at the time I wanted to play. I look online and wished there was an application like this but couldn't find it. So I made it! This application allows people to search for places to play (courts) by city. Then they can upvote courts and select a specific court to learn more about the court. Users can also register an account and can add a court that has not yet been uploaded. Users can create an account and their passwords are securely encrypted. I also learned about using AWS S3 to store images and allow users to upload images directly from their computer. I also implemented several features of the Google Maps API.

Draftstone

Custom Hearthstone Draft Application

Project Overview

This is the largest and must full-scale application in my portfolio. This application was created over many months by myself and a friend from my cohort at Galvanize. Draftstone is a full stack web application with heavy logic in the background. This project was inspired when a friend and I were playing Hearthstone (the #1 worldwide electronic card game) and we wanted to draft decks to play against each other. However, that feature is not available in the game... so I decided to make it. Not only this, but many other features have been added including: creating user accounts, creating custom draft filters, saving game modes, publishing game modes to the Draftstone community, favoriting/voting on community games, trending games modes, importing user card collections, exporting draft decks directly to Hearthstone, and much more! Draftstone is currently in Beta and many more features are on our Trello board and will be added as we get feedback from the Draftstone Community.

What I Learned

I have learned so much in this process it is impossible to detail. First, I have learned so much by having to work in a team. For example, we had to learn how to communicate our tasks and how to resolve merge conflicts in Github. We used various forms of storage including our database, sessions, and local storage. I learned many new things about security including authentication, hashing, and allowing secure password updates. I implemented Nodemailer in order to send the user an email with a random 6-digit temporary password when they forget their password and cannot login. I also implemented Google analytics for the first time to track all of our users and visits. I implemented social media sharing and tracking for the first time. Users can share and help market Draftstone on Facebook, Twitter, Google+, Reddit, and email. Finally, we learned so much by deploying to Amazon AWS and being in a production environment for the first time. I learned that just because everything is working in development doesn't mean it will in production. There was a lot of learning and problem-solving necessary to get our site up and running with AWS and the RDS instance of our postgres database. All of this learning was very eye-opening and invaluable.

Rentivate

Peer to Peer Item Rental Application

Project Overview

Rentivate was my Capstone project for my software development immersive program. The idea was that there are major sites to sell your used items (Craigslist, Ebay) or rent out your home (airBnB), but I was not aware of any major company for renting out your used items that go unused around the home. People all have items they spent a lot of money on that they only use once or twice a year; this application will allow them to collect cash instead of dust and for renters to have access to items they would not want or be able to purchase. Owners can upload and manage rentals, manage bookings, and decline/confirm reservations. Renters can browse all categories and rentals, reserve and view their rentals/bookings, make payments directly through Stripe, and more! There is a lot more to implement including a two way review/rating system between owners and renters, and then the further goal to eventually implement the option to allow a third party to manage inventory items so an owner never has to deal with a renter directly.

What I Learned

In this project I spent a lot of time sharpening my React and Redux skills. I also learned a couple new technoligies including SASS to write more efficent and modular CSS, MongoDB for the non-relational database, and Mongoose as the ORM. I learned and implemented Javascript Web Tokens (JWT) for authentication. I spent a lot of time learning to improve validation and handling/displaying errors. I used AWS S3 for image storage/upload, the GoogleMaps API, and integrated with the Stripe API for the first time to securely receive payments. As the capstone to my program, when I look back at my first static site I built (the landing page) I see how far I have come and it is very encouraging and motivating.
